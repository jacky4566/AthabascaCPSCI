Minimaxing and Learning

This assignment is worth 30% of your final grade. Submit it after you have completed Unit 9.

Game 1: Minimaxing (50 marks)

The objective of this program is to implement a “Connect 4” board game with two players: a user and a computer. The board should have 7 columns and 6 rows. The user plays with white chips and the computer plays with black chips.

Suppose that the board is placed in a vertical position and the players introduce chips at the top of a chosen column. The board is under the influence of gravity, and each chip always falls to the lowest available row in the column. The player with white chips goes first, and then the players take turns placing one chip at a time into the board game. The first player who is successful in creating a connected line of 4 chips on the board—horizontally, vertically or diagonally—is the winner. [Figure 1 illustrates four white chips in a row diagonally, resulting in a win.] If the board is full and no player has won, then the game ends in a tie.

Note

It is recommended that you implement alpha–beta pruning and set a limit for depth cut off for the minimax game tree search for the black chips (computer). For example, if you specify a depth cut off of 4, the player with the black chips is only allowed to build the game tree 4 rows deep into the board.

Game 2: Learning (50 marks)

The objective of this program is to implement a learning technique in a game of your choice. You can either implement a simple new game or use one you have already implemented in Assignment 1 or Assignment 2. You may use the source code from a game you find on the Internet as long as you clearly reference the original creator and source and you implement the learning algorithm by yourself.

Choose any learning algorithm from the ones we examined in this course (hill climbing, N-gram, ID3, Q-learning, etc.). If you are interested in implementing a learning algorithm not covered in the course, please contact your instructor to obtain permission first.

Note

Extra credits up to 15% will be given to students who choose to implement a new game. No mark will be deduced for those who choose to use one of their games implemented in previous assignments.

Submission Requirements

For each game, submit the following files:

  • your Java source code files
  • an executable file for your game (.jar, .exe., etc.)
  • any sound files used in your program
  • any textures or images used in your program
  • a description file containing the following information:
    • a description of the logic and design of your game
    • instructions on how to compile/run your game
    • a list of any bugs or problems in your game

Marking

Your overall grade for each game is based on the following:

  • fulfillment of the requirement that your program be a working game (20 marks)
  • creativity (5 marks)
  • inclusion of a description file that contains details about the design and implementation of your game. (20 marks)
  • Good code design and style: is your coding consistent, readable, commented and modularized? (5 marks)
